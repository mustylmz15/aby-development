📊 Durum Kontrol Komutları

git status

git status
Ne yapar: Mevcut repository durumunu gösterir
Gösterdikleri:
  • Hangi dosyalar değişmiş
  • Hangi dosyalar staged (commit'e hazır)
  • Hangi dosyalar untracked (takip edilmiyor)
  • Branch bilgisi
Örnek çıktı: On branch main Your branch is up to date with 'origin/main'. Changes not staged for commit: modified: ABY-Portal/src/App.vue

git log

git log --oneline -5
git log --graph --oneline
git log --author="İsim"
Ne yapar: Commit geçmişini gösterir
Seçenekler:
  • --oneline: Kısa format
  • -5: Son 5 commit
  • --graph: Görsel ağaç formatı
  • --author: Belirli kişinin commit'leri

📝 Değişiklik Yönetimi

git add

git add .
git add README.md
git add ABY-Portal/src/
git add *.vue
Ne yapar: Dosyaları "staging area"ya ekler (commit'e hazırlar)
Kullanım örnekleri:
  • .: Tüm değişiklikleri ekle
  • dosya.txt: Sadece belirli dosyayı ekle
  • klasor/: Sadece klasörü ekle
  • *.vue: Sadece Vue dosyalarını ekle

git commit

git commit -m "feat: yeni özellik eklendi"
git commit -m "fix: ABY-Portal login düzeltmesi"
git commit --amend -m "Mesajı düzelt"
Ne yapar: Staging area'daki değişiklikleri kalıcı olarak kaydeder
ABY Project commit standartları:
  • feat: - Yeni özellik
  • fix: - Hata düzeltmesi
  • docs: - Dokümantasyon
  • style: - Kod formatı
  • refactor: - Kod iyileştirme

🔄 Uzak Repository İşlemleri

git pull

git pull origin main
git pull
Ne yapar: Uzak repository'deki değişiklikleri indirir ve birleştirir
Ne zaman kullanılır: Günün başında, arkadaşınızın değişikliklerini almak için

git push

git push origin main
git push origin feature/yeni-dal
git push -u origin main
Ne yapar: Yerel commit'leri uzak repository'ye gönderir
Seçenekler:
  • origin main: Ana dala gönder
  • -u: İlk kez gönderirken upstream ayarla

git fetch

git fetch origin
Ne yapar: Uzak repository'deki değişiklikleri indirir ama birleştirmez
Fark: pull = fetch + merge

🌿 Branch (Dal) Yönetimi

git branch

git branch
git branch -a
git branch feature/yeni-ozellik
git branch -d dal-adi
Ne yapar: Dal yönetimi işlemleri
Seçenekler:
  • -a: Tüm dalları listele
  • dal-adi: Yeni dal oluştur
  • -d: Dal sil

git checkout / git switch

git checkout main
git checkout -b feature/dashboard
git switch main
git switch -c feature/portal
Ne yapar: Dallar arası geçiş yapar
Modern komut: git switch (Git 2.23+)
Seçenekler:
  • -b / -c: Yeni dal oluştur ve geç
ABY Project'te örnek: git checkout -b feature/aby-portal-update # Portal'da değişiklik yap git add . git commit -m "feat: ABY-Portal navigation güncellendi" git push origin feature/aby-portal-update

🔍 İnceleme ve Karşılaştırma

git diff

git diff
git diff --staged
git diff HEAD~1
git diff main..feature
Ne yapar: Değişiklikleri karşılaştırır
Seçenekler:
  • git diff: Working directory vs staged
  • --staged: Staged vs son commit
  • HEAD~1: Son commit vs bir önceki
  • main..feature: İki dal arasındaki fark

git show

git show HEAD
git show cd7f341
Ne yapar: Commit detaylarını gösterir
Kullanım: HEAD = son commit, hash ile belirli commit

🔄 Geri Alma İşlemleri

git reset

git reset --soft HEAD~1
git reset --mixed HEAD~1
git reset --hard HEAD~1
Ne yapar: Commit'leri geri alır
Seçenekler:
  • --soft: Commit'i geri al, dosyalar staged kalsın
  • --mixed: Commit'i geri al, dosyalar unstaged
  • --hard: Commit'i geri al, dosyalar da geri gelsin
DİKKAT: --hard tüm değişiklikleri siler!

git revert

git revert HEAD
Ne yapar: Yeni bir commit oluşturarak önceki commit'i geri alır
Güvenli: Geçmişi silmez, yeni commit ekler

git checkout (dosya geri alma)

git checkout .
git checkout -- ABY-Portal/src/App.vue
Ne yapar: Dosya değişikliklerini geri alır
Kullanım: . = tüm dosyalar, dosya adı = belirli dosya
